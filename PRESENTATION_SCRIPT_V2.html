Section 3

Pipeline — latency budget + agent architecture ~45s

🖥 Screen: landing page — Pipeline section, let animation play
"Latency budget: Whisper transcription — 150ms. Google Translate fast packet — under 100ms, sent immediately. Llama 3.3 70B style refinement — under 500ms. Romaji — 5ms. Total: under 800 milliseconds. The two-pass split is why it feels instant — the user reads the translation before the LLM even finishes."
"Suggestions run deferred — never block translation from appearing. Three agents, distinct roles: the Analyst queries Qdrant per-speaker vector memory and reads the emotional register. The Strategist decides reply type — agreement, question, reaction, game callout. The Writer generates three style-matched Japanese replies. Each one is immediately pre-synthesized via edge-tts — so 'Bot Speaks' has zero extra delay."
"Separate semaphores for translations and suggestions — a slow agent run for speaker A can never block a translation packet from speaker B."
Reference only — don't speak unless asked Streaming LLM: translation and romaji are complete units — partial Japanese mid-sentence is more confusing than the 200ms wait. Streaming IS used for TTS: audio starts before the full MP3 is ready, cutting perceived TTS latency by ~half.

3 agents vs 1 prompt: Qdrant retrieval, reply-type decision, and Japanese generation are different reasoning tasks with different inputs. The Strategist's structured brief forces the Writer on topic rather than free-associating from raw transcript.

Section 4

AMD MI300X — why the hardware matters ~25s

🖥 Screen: AMD section — hero banner + comparison table
"Llama 3.3 70B at full FP16 is 140 gigabytes. An H100 has 80. The MI300X has 192 gigabytes of HBM3 — the entire model fits on one GPU, no quantization, no sharding."
— pause —
"That matters for Japanese. Honorifics, register shifts — they live in the tail of the probability distribution. INT4 quantization truncates that tail. Full FP16 is a quality requirement, not a hardware luxury."
⚠ Say "quality requirement" not "nice to have." Show you made a deliberate engineering tradeoff, not that you got lucky with hardware.

Architecture — systems thinking ~45s

🖥 Screen: Engineering section — solo build banner
"Five runtimes in production: TypeScript overlay, Python FastAPI server, Rust Tauri backend, JavaScript Discord bot, GLSL shaders on this page. Built solo, 7 days."
👆 Scroll to 3-step architecture diagram
"Layer one — Node.js captures each speaker as a separate Opus stream. Mix the audio before transcription and you lose speaker identity entirely. Each stream forwards over an SSH tunnel — zero public ports, identical behavior in dev and production."
"Layer two — AMD MI300X: Whisper transcribes with word-level timestamps, two-pass fires immediately. vLLM calls go through a persistent TCP session — one reused connection, avoids 20-50ms handshake overhead on every call."
"Layer three — Tauri renders the JSON packet. Transparent, always-on-top, native Rust window. Card updates in-place on the refined packet — no re-render flash."

Section 6

Engineering decisions — the tradeoffs ~35s

🖥 Screen: Engineering decision cards (6 cards)
"Six decisions where the tradeoff was non-obvious:"
01 openai-whisper, not WhisperX WhisperX uses CTranslate2 for acceleration — CTranslate2 has no ROCm backend. PyTorch-native openai-whisper runs directly on MI300X with full GPU utilization. WhisperX would have fallen back to CPU on this hardware, which is 10-20× slower.
02 Tauri, not Electron 50 megabyte binary vs 200 megabyte. Native always-on-top with no flicker. Rust memory safety as the process boundary. For an overlay that runs during competitive gaming, frame-time impact matters.
03 SSH tunnel, not a public WebSocket endpoint No ingress rules, no TLS certificates, no token rotation. The AMD cloud instance has zero open ports. Identical behavior in local development and production — one flag change.
04 Per-speaker Opus streams, not mixed audio Discord's native API gives per-user PCM. Mixing destroys speaker identity before transcription. Separate streams let the UI attribute words to faces. It also allows per-member pipeline gating — exclude non-Japanese speakers before audio ever hits the GPU.
05 vLLM GPU util cap at 80% Default 100% leaves zero VRAM for Whisper. 80% reserves ~38GB — enough to run the full Whisper model without OOM. This is a single flag: --gpu-memory-utilization 0.80. The default would silently fail at runtime.
06 pykakasi, not MeCab MeCab requires a native shared library compiled against the OS and dictionary version. In a Docker container on ROCm with non-standard Python, that's an hour of dependency hell. pykakasi is pure Python — pip install, done. The romaji quality difference is negligible for pronunciation guidance.
⚠ For each card: read the title, pause briefly, then deliver the reasoning in one clean sentence. Don't rush through them.
